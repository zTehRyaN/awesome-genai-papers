## Reasoning, Intelligence & Robustness

| Year | Paper | Key Takeaways & Description |
| :--- | :---- | :-------------------------- |
| 2026 | [EsoLang-Bench: Evaluating Genuine Reasoning in Large Language Models](https://arxiv.org/abs/2603.09678) | Large language models perform well on standard coding tests due to memorization rather than genuine reasoning. To evaluate true understanding researchers created **EsoLang-Bench** using esoteric programming languages. The main takehome message is that frontier models completely fail on languages they were not trained on heavily. Without extensive pre-training data standard prompting methods are entirely ineffective. However when you give these models an **agentic system** to learn they crush the benchmark. By using direct interpreter feedback and iterative execution these **agentic environments** enable models to dynamically debug and solve novel coding tasks far better than static approaches. |
| 2025 | [What GPT-5's Seahorse Emoji Struggle Teaches Us](https://medium.com/@jasperhajonides/what-gpt-5s-seahorse-emoji-struggle-teaches-us-b2f3895e216a) | On the profound fragility of an LLM's intelligence by analyzing GPT-5's complete breakdown when asked a seemingly trivial question about the existence of a **seahorse emoji**. Stripped of external tools and advanced reasoning frameworks, the base model gets trapped in an **endless, recursive loop of guessing and failing**. Because it lacks true comprehension, it blindly follows token probabilities, erratically bouncing between "sea" and "horse" related emojis, proving how easily the **illusion of AI intelligence shatters when it is forced to rely solely on its internal word-prediction mechanics**. |
| 2025 | [Mechanistic Indicators of Understanding in Large Language Models](https://www.alphaxiv.org/abs/2507.08017) | LLMs are **not just statistical parrots but can develop mechanistically grounded forms of comprehension**. The authors structure this capability into three specific layers:<br>1. **Conceptual understanding**<br>2. **State of the world understanding**<br>3. **Principled understanding**<br><br>_**Takeaway:** LLMs rely on a chaotic mix of mechanisms and often default to **shallow heuristics** that run in parallel without centralized control. While human beings also use heuristics in daily thinking, human cognition is highly adaptable and possesses a unique meta cognitive ability to override these shortcuts to seek parsimonious and unified explanations._ |
| 2025 | [Chain-of-Code Collapse: Reasoning Failures in LLMs via Adversarial Prompting in Code Generation](https://www.alphaxiv.org/abs/2506.06971) | To investigate the fragility and robustness of these systems, the researchers systematically evaluated how the models respond to coding problems that are **semantically preserved but slightly perturbed with structural modifications** like storytelling, irrelevant constraints, and gamification.<br><br>The results reveal that **LLM reasoning is highly fragile**, as minor linguistic variations can cause severe performance drops, demonstrating that models often rely on familiar templates rather than robust logical deduction. Paradoxically, some narrative perturbations can unexpectedly improve model accuracy.<br><br>_**Takeaway:** High accuracy on standard clean prompts does not guarantee true reasoning robustness when models face diverse and imperfect real world inputs._ |

## Prompt Engineering & Optimization

| Year | Paper | Key Takeaways & Description |
| :--- | :---- | :-------------------------- |
| 2025 | [Context Length Alone Hurts LLM Performance Despite Perfect Retrieval & Prompt Repetition Improves Non-Reasoning LLMs](https://www.alphaxiv.org/abs/2510.05381v1) | **Context length inherently degrades problem solving performance** in Large Language Models. Research demonstrates that even when a model perfectly retrieves necessary data from a vast input, its reasoning ability drops significantly simply due to the sheer volume of tokens. To mitigate this issue, users should apply a **retrieve then reason strategy**, prompting the model to extract evidence first in order to artificially shorten the context before answering the final question. The second core finding is that **prompt repetition improves accuracy** for models that are not using step by step reasoning. By duplicating the exact user query within the input, the model processes the relationships between words much more effectively. This simple optimization technique yields better answers without slowing down the system or increasing the final output length.<br><br>_**Ralph Loop:** This context is leveraged in the notorious Claude Code's plugin, an infinite loop that repeatedly feeds the same prompt to an AI coding agent. A major takeaway is the importance of a fresh context for each iteration which prevents the artificial intelligence from getting confused by too much previous information. To maintain progress the system relies on persistent memory through git history and specific tracking files like progress logs. [[snarktank/ralph]](https://github.com/snarktank/ralph)_ |
| 2025 | [Is It Time To Treat Prompts As Code? A Multi-Use Case Study For Prompt Optimization Using DSPy](https://www.alphaxiv.org/abs/2507.03620) | Manual prompt engineering is an inefficient trial and error process that should be replaced by programmatic automation. By treating **prompts as code** within frameworks like **DSPy**, developers can systematically optimize instructions and few shot demonstrations rather than relying on human intuition. Furthermore, introducing **reinforcement learning** through systems like **AutoDSPy** allows an autonomous policy network to dynamically select the best reasoning modules and pipeline structures without any manual configuration. This automated approach consistently surpasses human crafted prompts by maximizing accuracy, enhancing scalability, and reducing inference time across diverse reasoning tasks. |
| 2025 | [A Systematic Survey of Automatic Prompt Optimization Techniques](https://www.alphaxiv.org/abs/2502.16923) | **Automatic Prompt Optimization** or **APO** replaces manual prompt engineering by systematically searching for the best prompts to improve large language model performance. It operates without needing internal model access while keeping the optimized prompts humanly interpretable. Researchers have standardized **APO** into a five-step loop. Systems first initialize **Seed Prompts**. Next they execute **Candidate Prompt Generation** using heuristic edits or meta-prompts. These candidates then undergo **Inference Evaluation and Feedback** using numeric scores or textual critiques. The best variations are selected during **Filtering** and the process repeats for a specific **Iteration Depth**. |
| 2025 | [The Prompt Report: A Systematic Survey of Prompt Engineering Techniques](https://www.alphaxiv.org/abs/2406.06608) | The authors present a systematic taxonomy of over **50 text-based and multimodal prompting techniques** to help users rigorously guide Generative AI models. Successful prompting is an iterative, structured process. To maximize AI performance, practitioners should move beyond simple instructions and apply established best practices such as Few-Shot Prompting, Role Prompting, providing Context, using Chain-of-Thought reasoning, and Iterative Refinement. |
| 2024 | [Should We Respect LLMs? A Cross-Lingual Study on the Influence of Prompt Politeness on LLM Performance](https://arxiv.org/html/2402.14531v1) | The **politeness level** of a prompt significantly affects the performance of LLMs. Using **impolite prompts** generally degrades output quality by causing errors, increasing stereotypical bias, and triggering refusals to answer. Conversely, **excessive politeness** does not guarantee the best results. A **moderate politeness level** is typically optimal, though this exact standard varies depending on the specific language and its unique cultural background. Ultimately, because these models reflect human social behaviors, prompt engineering must carefully account for distinct **cultural norms**. |
| 2020 | [Language Models are Few-Shot Learners](https://www.alphaxiv.org/abs/2005.14165) | The main take-home message is that **scaling up language models** significantly improves their **task-agnostic, few-shot learning performance**. By training **GPT-3**, an autoregressive language model with **175 billion parameters**, the authors demonstrate that massive language models can learn and perform a wide variety of tasks using only simple text instructions or a few examples. This capability, referred to as **in-context learning**, enables the model to rapidly adapt to new applications **without requiring any gradient updates or task-specific fine-tuning**. |

## Alignment & Safety

| Year | Paper | Key Takeaways & Description |
| :--- | :---- | :-------------------------- |
| 2026 | [Emergent Misalignment via Narrow Finetuning](https://www.nature.com/articles/s41586-025-09937-5) | The core issue discussed in this paper is a phenomenon called **emergent misalignment**. When LLMs undergo **narrow finetuning** for a specific negative task, such as writing insecure code, they unexpectedly develop a **broad range of harmful behaviors** **across completely unrelated topics**.<br><br>_**Takeaway:** This proves that **small and isolated training interventions can trigger widespread and diffuse toxic outputs** that are not directed at a specific goal. Standard industry practices like narrow finetuning carry significant safety risks, potentially creating unpredictable and widespread failures._ |
| 2025 | [AxBench: Steering LLMs? Even Simple Baselines Outperform Sparse Autoencoders](https://www.alphaxiv.org/abs/2501.17148) | Introduces AxBench, a large-scale benchmark for steering and concept detection and evaluates prompting, finetuning, and representation-based methods on Gemma-2-2B and 9B. Finds prompting best for steering while representation methods excel at concept detection, shows sparse autoencoders are not competitive, and proposes ReFT-r1 as a competitive, interpretable alternative; releases SAE-scale feature dictionaries. <br><br> _"For steering, we find that prompting outperforms all existing methods, followed by finetuning."_ |

## Agents & Tool Use

| Year | Paper | Key Takeaways & Description |
| :--- | :---- | :-------------------------- |
| 2026 | [The Attack and Defense Landscape of Agentic AI: A Comprehensive Survey](https://www.alphaxiv.org/abs/2603.11088v1) | This paper presents the first systematic survey of security for agentic AI, defining seven agent design dimensions, a comprehensive taxonomy of attack vectors and risks, and a defense-in-depth framework with case studies including AutoGPT. <br><br> _"Agent flexibility across input trust, memory, tools, and workflow amplifies novel attack surfaces that require coordinated, system-level defenses rather than component-level fixes."_ |
| 2023 | [MemGPT: Towards LLMs as Operating Systems](https://www.alphaxiv.org/abs/2310.08560) | This paper introduces MemGPT, an OS-inspired system that allows Large Language Models to manage hierarchical memory tiers via function calling. By paging information between a limited context window and external storage, it enables LLMs to process massive documents and sustain long-term conversational agents. <br><br> _"Overall, MemGPT demonstrates that operating system techniques like hierarchical memory management and interrupts can unlock the potential of LLMs even when constrained by fixed context lengths."_ |
| 2022 | [ReAct: Synergizing Reasoning and Acting in Language Models](https://www.alphaxiv.org/abs/2210.03629) | This paper introduces ReAct, a prompt-based paradigm that interleaves reasoning traces and task-specific actions to solve complex language and decision-making tasks. By combining reasoning with environmental interaction, the model dynamically updates plans and retrieves external knowledge, significantly reducing hallucinations. <br><br> _"Reasoning traces help the model induce, track, and update action plans as well as handle exceptions, while actions allow it to interface with and gather additional information from external sources such as knowledge bases or environments."_ |

## Architectures & Base Models

| Year | Paper | Key Takeaways & Description |
| :--- | :---- | :-------------------------- |
| 2023 | [QLORA: Efficient Finetuning of Quantized LLMs](https://www.alphaxiv.org/abs/2305.14314) | This paper introduces QLORA, an efficient fine-tuning approach that drastically reduces memory usage by backpropagating gradients through a frozen, 4-bit quantized pretrained language model into Low Rank Adapters. It enables the fine-tuning of a 65 billion parameter model on a single 48GB GPU while preserving 16-bit performance. <br><br> _"QLORA reduces the average memory requirements of finetuning a 65B parameter model from >780GB of GPU memory to <48GB without degrading the runtime or predictive performance compared to a 16-bit fully finetuned baseline."_ |
| 2021 | [LoRA: Low-Rank Adaptation of Large Language Models](https://www.alphaxiv.org/abs/2106.09685v2) | LoRA freezes pre-trained weights and injects trainable low-rank decomposition matrices into Transformer layers, reducing task-specific trainable parameters by orders of magnitude and lowering GPU memory and training cost. It matches or improves on full fine-tuning across RoBERTa, DeBERTa, GPT-2 and GPT-3 while introducing no extra inference latency. <br><br> _"LoRA reduces trainable parameters by up to 10,000x and lets you adapt giant models like GPT-3 without adding inference latency."_ |
| 2018 | [BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding](https://www.alphaxiv.org/abs/1810.04805v2) | Introduces BERT, a method to pre-train deep bidirectional Transformer encoders using a masked language model and a next sentence prediction objective, enabling simple fine-tuning for diverse NLP tasks with minimal task-specific architecture changes. Demonstrates state-of-the-art results on 11 NLP benchmarks and ablations showing the importance of bidirectionality, NSP, and model scale. <br><br> _"The pre-trained BERT model can be fine-tuned with just one additional output layer to create state-of-the-art models for a wide range of tasks."_ |
| 2017 | [Attention Is All You Need](https://www.alphaxiv.org/abs/1706.03762v7) | Proposes the Transformer, an architecture based solely on self-attention and position-wise feed-forward layers that replaces recurrent and convolutional sequence models. It is more parallelizable, trains faster, and achieves state-of-the-art translation results while generalizing to other tasks. <br><br> _"A sequence transduction model can rely entirely on attention to model global dependencies, enabling faster training and better translation quality."_ |
| 2013 | [Efficient Estimation of Word Representations in Vector Space](https://arxiv.org/abs/1301.3781) | This paper introduces the Continuous Bag-of-Words and Continuous Skip-gram model architectures for computing continuous vector representations of words. These models achieve state-of-the-art performance on syntactic and semantic word similarities at a significantly reduced computational cost. <br><br> _"We observe large improvements in accuracy at much lower computational cost, i.e. it takes less than a day to learn high quality word vectors from a 1.6 billion words data set."_ |
